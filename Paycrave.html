Paycrave

Use the Prototype

Summary

Food trucks are a great place to get lunch in a city, unfortunately there is no unifed way to pre-order, read reviews, or pay with a credit card. This mobile application will provide users a way to view these trucks on a map and submit orders.

Explanation

For this project, I was given the style tile for the company and the task of creating this application so that users could search for nearby trucks, view and create reviews, view and order from menus, and pay by credit card.

Problem

Some research needed to be done to determine what was important to people who would possibly use this application. I created user stories for three potential people. A college student wants to find the least expensive food truck, wants to sort out the types of food he doesn't like, and wants a way to repeat an order. A patrolling police officer wants the fastest way to check out and what is nearest him. An office worker wants to view food trucks with high ratings.

Solution

I ran competitive analysis on Lyft and Uber taxi services, designed a user flow diagram, and ran a few user tests. My user tests determined what features are most important and I found that the priority would be to find food trucks near the user, so this should be the opening screen. Then I tested different pin options. I found that the most useful information to have on the pin head was the average rating which would be relatd to the color of the pin with the darkest pin being the best rated.

Results

I ran some tests on navigation and flow. I asked testers to order a specified list of items and found that the most difficult part for the majority of users was finding the specified food truck from a list. I determined that I needed more padding around the trucks and that the truck's name should be larger. I also ran a test to see if users would be able to make a repeat order from a history of orders.

I also did a Peek test and found that my testing user generally liked the design and layout. "I think this is a brilliant app. I would definitely use it, endorse it, and share it."

Conclusion

This project was useful for me to see how size is incredibly important while designing for mobile. I found that when testing the clickable prototype on a phone I had to increase the size of nearly all of my text. My original design flow changed after I was implementing features. Images are extremely useful. Overall I found it slightly difficult to determine how much I wanted to follow the current tradition of flow versus create something new.
